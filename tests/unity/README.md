# WorldZones Unity Test Suite

Unity test project for WorldZones world generation that requires Unity runtime.

## Purpose

This Unity project hosts tests that need to call Unity's native functions:
- `Mathf.PerlinNoise()` - Unity's Perlin noise implementation (matches Valheim)
- Other Unity-specific APIs as needed

## Architecture Decision

**Why Unity Test Suite instead of Lookup Table?**

Previously we tried:
1. Extract Unity Perlin values → Lookup table → Test in C#
2. Problems: Large 2.9MB file, still not 100% accurate, maintenance burden

New approach:
1. Run tests directly in Unity runtime (headless)
2. Call `Mathf.PerlinNoise()` natively
3. No lookup table needed
4. Guaranteed accuracy

## Running Tests

### CLI (Headless)

```powershell
# From repository root
.\scripts\Run-UnityTestSuite.ps1
```

Or manually:
```powershell
& "C:\Program Files\Unity\Hub\Editor\6000.3.8f1\Editor\Unity.exe" `
    -runTests `
    -batchmode `
    -nographics `
    -projectPath "C:\Users\dangreen\projects\valheim\worldzones\tests\unity" `
    -testPlatform PlayMode `
    -testResults "TestResults.xml"
```

### Unity Editor (Optional)

1. Open Unity Hub
2. Add project: `tests/unity`
3. Open project
4. Window → General → Test Runner
5. Run PlayMode tests

## .NET Conventions

This project follows .NET naming conventions where possible:
- **PascalCase** for files: `UnityPerlinNoiseTests.cs`
- **PascalCase** for types: `WorldGeneratorTests`
- **Namespaces**: `WorldZones.WorldGen.Tests`

Unity-specific elements use Unity conventions:
- `.asmdef` files for assembly definitions
- `.meta` files (auto-generated by Unity)

## Integration with CI/CD

The headless test runner is CI/CD friendly:
- No GUI required
- Exit codes: 0 = pass, non-zero = fail
- XML test results output
- Can run in Docker/containers with Unity installed
